1. Introduction & Philosophy

1.1 What is TextTool?

TextTool is a hybrid text processing environment that bridges the gap between command-line efficiency and visual editing. It’s designed for professionals who work with text data but need both the precision of scripting and the intuition of visual feedback.

1.2 Design Principles

  • Dual-Mode Operation: Every change is available via both CLI and GUI
  • Non-Destructive Workflow: Full revert capability at every step
  • Progressive Disclosure: Basic functions first, advanced features on demand
  • Context Awareness: Live View understands your current operation state
  • Extensibility: Plugin architecture for custom operations

1.3 Target Users

  • Developers: Code refactoring, log analysis, data transformation
  • Data Analysts: Data cleaning, format conversion, pattern extraction
  • System Administrators: Log processing, configuration management
  • Content Editors: Bulk text manipulation, template processing
  • Researchers: Text mining, data extraction, pattern analysis

1.4 Key Differentiators

  • Real-time Sync: CLI commands instantly reflect in Live View and vice versa
  • Intelligent Selection: Context-aware selection with visual feedback
  • Multi-format Support: Text files, Excel, clipboard, and direct input
  • Session Persistence: Command history and state preserved across sessions
  • Professional Regex: Full PCRE-compatible regex with special placeholders

4. Complete Command Reference

4.1 Core Operations

load - Content Loading

Purpose: Import text from various sources

Full Syntax:

load [file_path] [--encoding ENCODING]

Parameters:

  • file_path: Path to text file (optional)
  • --encoding: Force specific encoding (utf-8, latin-1, etc.)

Behavior:

  • No arguments: Load from system clipboard
  • File path: Load specified file with auto-encoding detection
  • Clipboard fallback: If clipboard empty, check for file paths in clipboard

Examples:

# Basic file loading
load "C:\data\logs\app.log"
load "../relative/path/file.txt"
load "file with spaces.txt"

# Clipboard operations
load  # Load text from clipboard
load  # If text selected in file explorer, load that file

# Encoding specification
load "file.txt" --encoding utf-8
load "legacy_file.txt" --encoding latin-1

Technical Details:

  • Encoding detection: UTF-8 → Latin-1 fallback
  • Line ending normalization: \r\n → \n
  • Memory mapping for large files
  • Automatic directory creation for save paths

show - Content Display

Purpose: Filter and display text with various criteria

Full Syntax:

show [pattern] [> output_target]

Parameters:

  • pattern: String or regex pattern to filter lines
  • > output_target: Redirect output to file or clipboard

Pattern Types:

  • Simple string: show "error"
  • Regex pattern: show "^ERROR.*FAILED$"
  • Multiple patterns: show "error OR warning OR critical"
  • Negation: show "!debug" (must use with other patterns)

Output Redirection:

show "error" > "errors.txt"    # Save to file
show "warning" >              # Copy to clipboard
show > "all_content.txt"      # Save all content

Advanced Examples:

# Complex pattern matching
show "error.*database OR warning.*memory"
show "(ERROR|FATAL).*[0-9]{4}"
show "!DEBUG.*!TEST"  # Exclude debug and test lines

# Combined operations
show "error" > | select "2023" | show "database"

select - Line Filtering

Purpose: Permanently filter lines based on criteria

Full Syntax:

select <pattern> [case_sensitive] [inverse]

Special Placeholders:

  • [pipe] = |
  • [doublequote] = "
  • [quote] = '
  • [tab] = tab character
  • [spaces] = one or more whitespace characters

Examples:

# Basic selection
select "error"
select "error OR warning"
select "!debug"  # Exclude debug lines

# Case handling
select "Error" case_sensitive
select "ERROR"  # case insensitive by default

# Complex patterns with placeholders
select "config[spaces]=[spaces]"
select "name[doublequote]"
select "column[tab]value"

# Regex patterns
select "^[A-Z].*[.!]$"  # Complete sentences
select "\d{3}-\d{2}-\d{4}"  # Social security numbers

replace - Text Transformation

Purpose: Find and replace text with regex support

Full Syntax:

replace "search_pattern" "replacement" [case_sensitive] [regex_flags]

Replacement Special Sequences:

  • \1, \2, … - Capture group references
  • \0 - Entire matched text
  • \n - Newline in replacement
  • \t - Tab in replacement

Examples:

# Simple replacement
replace "cat" "dog"
replace "colour" "color"

# Regex with capture groups
replace "(\d{2})-(\d{2})-(\d{4})" "\3/\2/\1"
replace "Mr\. (\w+)" "Dear \1"
replace "(\w+)=(\w+)" "setting:\1 value:\2"

# Complex transformations
replace ".*(error|warning).*" "ALERT: \1"
replace "(\d+),(\d+)" "\1.\2"  # Comma to decimal

# Using \0 for wrapping
replace "error" "[ERROR: \0]"
replace "\d+" "NUMBER(\0)"

4.2 File Operations

save - Content Persistence

Purpose: Save current text to file system

Full Syntax:

save [file_path] [--backup] [--append]

Behavior:

  • No arguments: Overwrite original loaded file
  • With path: Save to new location
  • Auto-directory: Creates parent directories if needed

Examples:

# Basic save operations
save                    # Overwrite original
save "output.txt"       # Save to new file
save "../backup/data.log"  # Save with path

# Advanced options
save "output.txt" --backup  # Create backup of original
save "log.txt" --append     # Append to existing file

4.3 History & State Management

revert - Undo Operation

Purpose: Restore previous state

Behavior:

  • Single-level undo for most operations
  • Preserves all changes before last operation
  • Works with: replace, select, delete, sort, unique

Usage:

# After making changes
replace "old" "new"
show  # Verify changes
revert  # Undo the replacement

unselect - Selection Reversion

Purpose: Restore full text while keeping modifications to selected lines

Technical Details:

  • Maintains selected_indices tracking
  • Reapplies modifications to original full text
  • Different from revert - preserves changes but restores context

Usage:

select "error"
replace "error" "RESOLVED"
unselect  # Show all lines with errors marked as RESOLVED

4.4 Advanced Operations (Enable with advanced)

bulk_replace - Batch Processing

Purpose: Multiple replacements from mapping file

Supported Formats:

  • Text files: Tab, space, or custom separated
  • Excel files: First two columns (search→replace)
  • Clipboard: Space-separated key-value pairs

Examples:

# Text file with tab separation
bulk_replace "replacements.txt" tab

# Excel file mapping
bulk_replace "mapping.xlsx"

# Clipboard content (format: "old1 new1 old2 new2")
bulk_replace

# With case sensitivity
bulk_replace "mapping.txt" space case_sensitive

Mapping File Formats:

# replacements.txt (tab separated)
old_text1    new_text1
old_text2    new_text2
"quoted text"    "replacement"

# Excel file (columns A and B)
A           B
"find1"     "replace1"
"find2"     "replace2"

extract_between - Delimited Extraction

Purpose: Extract text between start and end patterns

Full Syntax:

extract_between "start_pattern" "end_pattern" [case_sensitive] [inclusive]

Examples:

# HTML/XML extraction
extract_between "<div>" "</div>"
extract_between "<!--" "-->"
extract_between "<![CDATA[" "]]>"

# Code block extraction
extract_between "function " "end"
extract_between "if " "fi"
extract_between "{" "}"

# Log section extraction
extract_between "=== START ===" "=== END ==="
extract_between "BEGIN_TRANSACTION" "END_TRANSACTION"

conditional_replace - Context-Aware Replacement

Purpose: Replace only in lines matching specific criteria

Full Syntax:

conditional_replace "search" "replace" "target_pattern" [case_sensitive]

Examples:

# Only replace in error lines
conditional_replace "database" "DATABASE" "ERROR"

# Case-sensitive conditional replacement
conditional_replace "config" "CONFIG" "\[settings\]" case_sensitive

# Multiple conditions
conditional_replace "temp" "temporary" "error|warning"

6. Advanced Pattern Matching

6.1 Regular Expressions Mastery

Character Classes

# Basic classes
[aeiou]          # Any vowel
[^aeiou]         # Any non-vowel
[a-z]            # Any lowercase letter
[A-Z]            # Any uppercase letter  
[0-9]            # Any digit
[a-zA-Z]         # Any letter
[0-9a-fA-F]      # Hexadecimal digit

# Predefined classes
\d               # Digit [0-9]
\D               # Non-digit [^0-9]
\w               # Word character [a-zA-Z0-9_]
\W               # Non-word character [^a-zA-Z0-9_]
\s               # Whitespace [ \t\r\n\f]
\S               # Non-whitespace [^ \t\r\n\f]

Quantifiers & Repetition

# Greedy quantifiers
a*               # Zero or more (as many as possible)
a+               # One or more (as many as possible)  
a?               # Zero or one
a{3}             # Exactly 3
a{3,}            # 3 or more
a{3,5}           # 3 to 5

# Lazy quantifiers
a*?              # Zero or more (as few as possible)
a+?              # One or more (as few as possible)
a??              # Zero or one (prefer zero)

Anchors & Boundaries

^                # Start of line
$                # End of line
\A               # Start of string
\Z               # End of string
\b               # Word boundary
\B               # Non-word boundary

Groups & Capturing

(expression)     # Capturing group
(?:expression)   # Non-capturing group
(?<name>exp)     # Named capturing group
\1, \2, ...      # Backreferences to groups

6.2 Practical Regex Patterns

Email Addresses

# Basic email
\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b

# With Unicode support
\b[\w.%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b

URLs

# HTTP/HTTPS URLs
https?://[^\s]+

# With domain validation
https?://(?:[-\w.]+)+(?:/[^\s]*)?

Dates

# Various date formats
\d{1,2}[/-]\d{1,2}[/-]\d{2,4}
\d{4}[/-]\d{1,2}[/-]\d{1,2}

Phone Numbers

# US format
\b\d{3}[-.]?\d{3}[-.]?\d{4}\b

# International
\+\d{1,3}[-.\s]?\(?\d{1,4}\)?[-.\s]?\d{1,4}[-.\s]?\d{1,9}

6.3 TextTool-Specific Patterns

Using Placeholders

8. Performance & Optimization

8.1 Large File Handling

Memory Management Strategies

# For files >100MB
highlight_toggle off  # Disable Live View highlighting

# Process in chunks
select "1-10000"      # Work with first 10k lines
# ... process ...
unselect              # Restore full context

# Use streaming operations
bulk_replace "mapping.txt"  # More efficient than multiple replace calls

Performance Tips

  • Use select first: Filter data before complex operations
  • Batch operations: Combine multiple replacements into bulk_replace
  • Avoid real-time highlighting: Disable for files >50,000 lines
  • Use efficient patterns: Specific regex vs. broad patterns

8.2 Optimization Techniques

Efficient Pattern Design

# Instead of broad patterns
.*error.*          # Slow - scans entire lines
error              # Fast - stops at first match

# Use anchors when possible
^error             # Only check line start
error$             # Only check line end

Command Chaining

# Instead of multiple commands
select "error"
replace "old" "new"
show

# Chain operations
select "error" | replace "old" "new" | show

8.3 Resource Monitoring

Memory Usage Indicators

  • Live View lag: UI updates slow indicates memory pressure
  • Command delay: Operations take longer than expected
  • High CPU: Python process using excessive CPU

Mitigation Strategies

# Reduce memory footprint
highlight_toggle off
select "1-5000"  # Work with subsets

# Use file-based operations
save "temp.txt"
load "temp.txt"  # Clears undo history, frees memory

9. Troubleshooting Master Guide

9.1 Common Error Messages

“Error: No file is loaded”

Causes:

  • Forgot to use load command
  • File path incorrect
  • Clipboard empty when loading from clipboard

Solutions:

# Load from file
load "path/to/file.txt"

# Load from clipboard
# First copy text to clipboard, then:
load

# Verify file exists
load "existing_file.txt"

“Error: Invalid regex pattern”

Causes:

  • Unescaped special characters
  • Unbalanced parentheses/brackets
  • Invalid quantifier usage

Solutions:

# Use cheat sheet for reference
cheat_sheet_regex

# Escape special characters
replace "file\.txt" "document.txt"

# Use placeholders for problematic chars
replace "text[doublequote]" "text[quote]"

“Error: No previous state to revert to”

Causes:

  • No operations performed yet
  • Already at initial state
  • Memory cleared by certain operations

Solutions:

  • Ensure at least one modification before revert
  • Use save before major changes as backup

9.2 Performance Issues

Slow Operations with Large Files

Symptoms:

  • Commands take several seconds
  • Live View becomes unresponsive
  • High memory usage

Solutions:

# Disable highlighting
highlight_toggle off

# Process in chunks
select "1-10000"
# ... work on chunk ...
unselect

# Use more specific patterns

Causes:

  • Window minimized or closed
  • Highlighting disabled
  • Sync issues between CLI and GUI

Solutions:

# Reopen Live View
liveview

# Enable highlighting
highlight_toggle on

# Force refresh
show  # Any command that updates display

Solutions:

Clipboard Integration Issues

Solutions:
